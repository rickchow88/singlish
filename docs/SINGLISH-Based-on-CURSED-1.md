## 3. Singaporean English (Singlish) Vocabulary

### 3.1 What is Singlish?

Singlish is an **English-based creole** spoken colloquially in Singapore. It's a mixture of English, Hokkien, Cantonese, Malay, and Tamil, with unique slang and syntax.

**Key Characteristics:**

- Borrowed from multiple languages (Hokkien, Cantonese, Malay, Tamil)
- Unique pronunciation and grammar
- Cultural identity marker for Singaporeans
- Informal contexts (avoided in formal settings)
- Some terms in Oxford English Dictionary

## 6. Cultural Considerations

### 6.1 Respecting Singlish Authenticity

**Do's:**

- Use terms that are actually used in Singapore
- Borrow from all source languages (Hokkien, Cantonese, Malay, Tamil)
- Keep grammar flexible (like real Singlish)
- Avoid forced/pure English terms
- Embrace the multilingual nature

**Don'ts:**

- Don't invent non-existent Singlish terms
- Avoid offensive terms (even if they exist in Singlish)
- Don't overuse particle words (lah, lor, meh) in code syntax
- Don't mix Singlish with formal programming jargon unnecessarily

### 6.2 Practical Programming Language

**Essential Features:**

- Clear, unambiguous syntax (despite Singlish vocabulary)
- Type safety
- Good error messages (in Singlish when helpful)
- Performance
- Tooling support (VS Code, debugging)
- Documentation in both Singlish and English

**Balance:**

- 80% Singlish keywords (for cultural identity)
- 20% standard programming terms (for clarity)
- Avoid "lah" in actual code syntax (use for comments/explanations only)

## 9. Challenges & Solutions

### 9.1 Technical Challenges

**Challenge 1: Disambiguation**
*Singlish term "can" has multiple meanings: ability vs permission*

**Solution:** Use explicit alternatives (kena vs boleh).

**Challenge 2: Multilingual Borrowings**
*Terms from Hokkien, Cantonese, Malay, Tamil mixed together*

**Solution:** Document source language in comments and accept all variants. Build flexible tokenizer.

**Challenge 3: Performance vs Readability**
*Singlish terms are longer (3-5 letters vs 2-3 English)*

**Solution:** Accept longer keywords for cultural authenticity. Optimize for parsing speed, not keyword length.

**Challenge 4: Tooling Adoption**
*Existing tools expect English keywords*

**Solution:** Create custom language server, formatter, linter. Provide VS Code extension.
