### STORY-002: Support Range Loops
Status: in_progress
Depends on: None

Description:
Implement Range Loop parsing and codegen support.

Issue:
Parser only supports C-style/while-style loops. Fails on 'loop i, v = all orders'.

Technical Spec:
- Update parseForStatement to handle 'range' or 'all' keywords.
- Update AST: Possibly add RangeLoopStatement or update ForStatement to support range semantics.
- Implement code generation for range loops.

Acceptance Criteria:
- [ ] Parser accepts 'loop i, v = all orders' syntax.
- [ ] Codegen produces valid Go 'for i, v := range orders' (or equivalent).
- [ ] Integration tests with range loops pass.
